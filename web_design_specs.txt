## 2. 特效与交互实现记录

### [特效 01] 视差滚动效果 (Parallax Scrolling)
- **目标**: 增加页面视觉深度。
- **实现方式**: 
  - 使用 CSS 属性 `background-attachment: fixed`。
  - 设置 `background-size: cover` 确保背景图填充。
  - 通过 `background-position: center` 保持视觉中心。
- **优点**: 纯 CSS 实现，性能极高，无 JS 抖动。
- **适用场景**: 落地页、欢迎页背景、品牌展示区。

### [特效 02] 全屏视频背景 (Fullscreen Video Background)
- **目标**: 营造沉浸式视觉体验，快速传达品牌氛围。
- **实现方式**: 
  - 使用 HTML5 <video> 标签，设置 `autoplay muted loop playsinline`。
  - 视频容器使用 `position: fixed` 固定在视口，z-index 设置为 -1。
  - 使用 `object-fit: cover` 确保视频在各种屏幕尺寸上完美填充。
  - 添加半透明遮罩层 (`background-color: rgba(0,0,0,0.4)`) 提升前景文字对比度。
- **关键属性**:
  - `min-width: 100%`, `min-height: 100%`: 确保视频始终填满容器。
  - `transform: translate(-50%, -50%)`: 保持视频居中。
- **适用场景**: 首页、产品登录页、活动专题页、品牌展示页。

### [特效 03] CSS3 滚动转换效果 (Scroll Transitions)
- **目标**: 当元素进入视口时触发优雅的入场动画，提升用户体验。
- **实现方式**: 
  - 使用 Intersection Observer API 检测元素是否进入视口。
  - 为元素添加初始状态类（如 `fade-in-up`），设置 `opacity: 0` 和初始 `transform`。
  - 当元素进入视口时，添加 `is-visible` 类触发 CSS transition。
- **关键属性**:
  - `scroll-behavior: smooth`: 平滑滚动效果。
  - `transition: opacity 0.6s ease-out, transform 0.6s ease-out`: 定义过渡属性和时长。
  - `transition-delay`: 为不同元素设置延迟，创造层次感。
- **动画类型**:
  - `fade-in-up`: 从下方淡入
  - `fade-in-left`: 从左侧淡入
  - `fade-in-right`: 从右侧淡入
  - `scale-in`: 缩放淡入
  - `rotate-in`: 旋转淡入
- **优点**: 性能优异，不影响 SEO，易于维护和扩展。
- **适用场景**: 产品列表、图片画廊、博客文章列表、特性展示页。

### [特效 04] 全屏覆盖响应式导航菜单 (Fullscreen Overlay Navigation)
- **目标**: 提供优雅的全屏导航体验，突出品牌形象。
- **实现方式**: 
  - 使用固定定位的导航栏，包含 Logo 和汉堡菜单按钮。
  - 全屏覆盖菜单使用 `position: fixed` 覆盖整个视口。
  - 通过添加/移除 `active` 类控制菜单的显示/隐藏。
  - 菜单项使用延迟动画，创造依次淡入的效果。
  - 汉堡菜单按钮通过 CSS transform 实现流畅的动画效果。
- **关键属性**:
  - `z-index`: 控制层级关系，确保菜单在最上层。
  - `opacity` 与 `visibility`: 平滑的淡入淡出效果。
  - `transition-delay`: 为不同菜单项设置延迟，创造层次感。
  - `transform: rotate()`: 实现汉堡菜单的动画变换。
- **交互特性**:
  - 点击汉堡菜单按钮切换菜单状态。
  - 点击菜单项自动关闭菜单。
  - 按 ESC 键关闭菜单。
  - 菜单打开时禁止页面滚动。
- **视觉设计**:
  - 渐变背景色，营造现代感。
  - 菜单项悬停效果，包含下划线动画。
  - 底部社交链接区域。
- **优点**: 视觉冲击力强，操作直观，适配各种设备。
- **适用场景**: 品牌官网、作品集、创意网站、活动专题页。

### [特效 05] Box Shadow 加载动画效果 (Box Shadow Loaders)
- **目标**: 提供优雅的加载状态提示，提升用户等待体验。
- **实现方式**: 
  - 利用 CSS box-shadow 属性创造动态阴影效果。
  - 结合 CSS @keyframes 实现平滑的动画过渡。
  - 使用 transform 属性实现旋转、缩放、位移等变换效果。
- **动画类型**:
  1. **脉冲阴影 (Pulse Shadow)**: 向外扩散的脉冲效果
  2. **旋转阴影 (Rotate Shadow)**: 带阴影旋转的方块
  3. **弹跳阴影 (Bounce Shadow)**: 上下弹跳的圆点
  4. **双阴影 (Double Shadow)**: 内外双层阴影旋转
  5. **呼吸阴影 (Breathe Shadow)**: 三层阴影呼吸效果
  6. **圆点阴影 (Dot Shadow)**: 三个圆点的跳跃动画
  7. **霓虹阴影 (Neon Shadow)**: 炫目的霓虹发光效果
  8. **条形阴影 (Bar Shadow)**: 音乐条式的波动效果
- **关键属性**:
  - `box-shadow`: 支持多层阴影，创造丰富的视觉效果
  - `@keyframes`: 定义动画的关键帧
  - `transform: rotate() / scale() / translateY()`: 实现各种变换效果
  - `animation`: 控制动画的播放
- **优点**: 
  - 纯 CSS 实现，无需 JavaScript
  - 性能优异，不影响页面加载
  - 易于自定义和扩展
  - 兼容性好，支持现代浏览器
- **适用场景**: 
  - 页面加载状态
  - 数据请求等待
  - 表单提交等待
  - 内容加载指示器

### [特效 06] CSS 反向悬停效果 (Reverse Hover Effects)
- **目标**: 当悬停某个元素时，使其他元素发生变化，创造独特的交互体验和视觉层次。
- **实现方式**: 
  - 使用 CSS 选择器 `.container:hover .item` 实现悬停容器时，所有子元素的变化。
  - 使用 `.container .item:hover` 覆盖被悬停元素的样式，使其保持突出状态。
  - 使用相邻兄弟选择器 `.item:hover + .item` 实现悬停时，相邻元素的变化。
  - 使用通用兄弟选择器 `.item:hover ~ .item` 实现悬停时，后续所有元素的变化。
  - 结合 transition 实现平滑的动画过渡。
- **效果类型**:
  1. **透明化其他元素**: 悬停某个元素时，其他元素变透明、缩小
  2. **相邻元素变化**: 悬停某个元素时，右边的相邻元素变色并移动
  3. **后续全部变化**: 悬停某个元素时，后面的所有元素都变色并旋转
  4. **卡片模糊效果**: 悬停某张卡片时，其他卡片变模糊
  5. **背景变化**: 悬停按钮时，整个容器背景变色
  6. **列表反向悬停**: 悬停列表项时，其他列表项变透明
  7. **文字散开**: 悬停单个字母时，其他字母变小变淡
  8. **字符变色**: 悬停单个字符时，该字符变色，其他变灰
- **关键属性**:
  - `transition`: 控制所有属性的平滑过渡
  - `opacity`: 实现透明化效果
  - `transform`: 实现缩放、位移、旋转等变形效果
  - `filter: blur()`: 实现模糊效果
  - `background`: 实现背景色变化
- **选择器技巧**:
  - 先对所有元素应用悬停效果
  - 再对当前悬停的元素应用特殊样式进行覆盖
  - 使用伪类选择器精确控制
- **优点**: 
  - 纯 CSS 实现，无需 JavaScript
  - 交互体验独特，令人印象深刻
  - 易于理解和维护
  - 兼容性好，支持现代浏览器
- **适用场景**: 
  - 产品展示网格
  - 导航菜单
  - 图片画廊
  - 特色功能介绍
  - 团队成员展示
  - 创意交互界面

### [特效 07] 垂直和水平对齐文本中心 (Text Centering)
- **目标**: 实现文本在容器中垂直和水平居中对齐，满足各种布局需求。
- **实现方式**: 
  - **Flexbox**: 使用 `display: flex` 配合 `justify-content: center` 和 `align-items: center`
  - **Grid**: 使用 `display: grid` 配合 `place-items: center`（最简洁的方式）
  - **Position + Transform**: 使用绝对定位配合 `top: 50%; left: 50%; transform: translate(-50%, -50%)`
  - **Table**: 使用 `display: table` 配合 `display: table-cell` 和 `vertical-align: middle`
  - **Line-height**: 设置行高等于容器高度（仅适用于单行文本）
  - **Absolute + Margin Auto**: 使用绝对定位配合 `margin: auto` 和设置四个方向都为 0
  - **Writing-mode**: 结合书写模式实现垂直文本居中
  - **Flexbox 多行**: 使用 Flexbox 对齐多行文本
  - **Grid 多行**: 使用 Grid 对齐多行文本
  - **Bootstrap 工具类**: 使用 Bootstrap 内置类 `d-flex justify-content-center align-items-center`
- **对齐方法**:
  1. **Flexbox**: 最常用且最灵活的方法，适合各种场景
  2. **Grid**: 最简洁的方法，一行代码搞定，适合现代浏览器
  3. **Position + Transform**: 兼容性好，适合需要精确控制定位的场景
  4. **Table**: 老式方法，适合需要兼容旧浏览器的场景
  5. **Line-height**: 仅适用于单行文本，简单直接
  6. **Absolute + Margin Auto**: 另一种绝对定位方式，需要明确容器尺寸
  7. **Writing-mode**: 专门用于垂直文本对齐
  8. **Flexbox 多行**: Flexbox 配合多行文本
  9. **Grid 多行**: Grid 配合多行文本
  10. **Bootstrap 工具类**: 使用框架类，快速实现
- **关键属性**:
  - `display: flex`: 启用 Flexbox 布局
  - `justify-content: center`: 水平居中对齐
  - `align-items: center`: 垂直居中对齐
  - `display: grid`: 启用 Grid 布局
  - `place-items: center`: 同时设置水平和垂直居中
  - `position: absolute`: 绝对定位
  - `top: 50%; left: 50%`: 将元素左上角定位到容器中心
  - `transform: translate(-50%, -50%)`: 将元素自身中心移动到容器中心
  - `line-height`: 设置行高
  - `margin: auto`: 自动外边距
- **优点**: 
  - 提供多种方案，满足不同场景需求
  - 纯 CSS 实现，无需 JavaScript
  - 兼容性良好，大部分方法支持现代浏览器
  - 易于理解和维护
- **适用场景**: 
  - 按钮文本居中
  - 卡片内容居中
  - Hero 区域标题居中
  - 模态框内容居中
  - 导航栏项目居中
  - 各种需要居中对齐的布局

### [特效 08] 问卷弹窗交互 (Questionnaire Modal)
- **目标**: 在用户完成注册登录后，优雅地弹出问卷收集用户信息
- **实现方式**: 
  - 使用 Bootstrap Modal 组件实现弹窗
  - 弹窗设置为 `backdrop: static` 和 `keyboard: false`，防止用户意外关闭
  - 弹窗内容区域设置 `max-height: 70vh` 和 `overflow-y: auto`，支持滚动
  - 使用 Thymeleaf 条件判断 `th:if="${!user.hasCompletedQuestionnaire}"` 控制是否显示弹窗
  - 问卷表单直接嵌入弹窗中，避免使用 iframe
- **问卷设计**:
  - **问题1**: 你是否具备计算机基础？（单选：零基础/略有了解/掌握常用电脑操作/专业基础良好）
  - **问题2**: 你是否了解过计算机编程语言？（单选：从未听说/仅听过名称/简单接触过/系统学习过1门及以上）
  - **问题3**: 目前你接触过的语言有哪些？（多选：Python/Java/C/C++/JavaScript/暂无/其他[输入框]）
  - **问题4**: 你最想优先学习的编程语言是？（单选：Python/Java/C/C++/JavaScript/Go/其他[输入框]）
  - **问题5**: 你学习编程语言的主要目的？（单选：兴趣爱好/学业要求/就业/提升技能/备考考证/其他[输入框]）
- **表单特性**:
  - 单选按钮使用 `<input type="radio" name="xxx" required>` 确保必填
  - 多选按钮使用 `<input type="checkbox" name="knownLanguages">`
  - "其他"选项使用 JavaScript 控制对应输入框的显示/隐藏
  - "暂无"选项与其他选项互斥
  - 提交按钮使用渐变色设计，悬停时有上浮和阴影效果
- **数据存储**:
  - User 模型新增字段：`hasCompletedQuestionnaire`, `computerBase`, `programmingKnowledge`, `knownLanguages`, `preferredLanguage`, `learningPurpose`, `otherKnownLanguage`, `otherPreferredLanguage`, `otherLearningPurpose`
  - 多选答案使用逗号分隔的字符串存储
- **UI 设计**:
  - 弹窗头部使用渐变背景（#667eea → #764ba2）
  - 选项卡片使用浅灰背景（#f8f9fa），悬停时加深
  - 输入框获得焦点时显示蓝色边框和阴影
  - 按钮使用同样的渐变色主题
- **优点**: 
  - 用户体验流畅，不会打断操作流程
  - 弹窗可滚动，适应不同屏幕尺寸
  - 表单验证完善，确保数据质量
  - 与整体设计风格保持一致
- **适用场景**: 
  - 用户注册后的信息收集
  - 产品使用前的需求调查
  - 用户偏好设置
  - 教育类产品的入学测试
